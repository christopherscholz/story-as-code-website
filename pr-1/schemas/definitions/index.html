2.1.2. Property tag.schema.json

Typeobject
RequiredNo
Additional propertiesAny type allowed
Defined intag.schema.json

Description: A tag definition — display name and description for a tag ID. Defining a tag is optional; elements may reference any tag ID and it will be treated as valid even without a matching definition here.

Examples:

id: protagonist
name: Protagonist
description: The central character whose journey drives the story.

id: magic
name: Magic
description: Related to the supernatural magic system of this world.

3.1.2. Property type.schema.json

Typeobject
RequiredNo
Additional propertiesAny type allowed
Defined intype.schema.json

Description: A type definition — defaults and required properties for a given element type. Defining a type is optional; elements may reference any type ID and it will be treated as valid even without a matching definition here.

Examples:

id: CHARACTER
name: Character
applies_to: NODE
description: A sentient being that can act, speak, and have relationships.
required_properties:
- age
defaults:
  alive: true

id: SPATIAL
name: Spatial Relationship
applies_to: EDGE
description: A spatial or containment relationship between two nodes.
defaults:
  directed: true
