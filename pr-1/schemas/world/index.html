2. Property time_system

Typeobject
RequiredNo
Additional propertiesAny type allowed
Defined intime-system.schema.json

Description: How time works in this world — its structure and calendar.

Examples:

id: linear
name: Linear Timeline
type: LINEAR
properties:
  epoch:
    name: The Founding
    description: Year zero begins with the founding of the First Kingdom
  units:
    major: cycles
    minor: phases
    phases_per_cycle: 8

id: iso-8601
name: Gregorian Calendar
type: LINEAR

id: dreamworld
name: Dreamworld
type: ABSTRACT
description: Time has no direction here; connections define sequence
properties:
  perception:
    observer_dependent: true
    description: Each dreamer experiences a different duration for the same event
  anchors:
    description: Recurring events that dreamers use to orient themselves in time
    examples:
    - the bell tolls
    - the tide turns
    - the sky fractures

4.1.2. Property edge.schema.json

Typeobject
RequiredNo
Additional propertiesAny type allowed
Defined inedge.schema.json

Description: A relationship between two nodes that persists over time.

Examples:

id: cave-adjacent-lighthouse
type: SPATIAL
name: adjacent-to
source: hidden-cave
target: lighthouse
directed: false

id: hero-in-dungeon
type: SPATIAL
name: located-in
source: hero
target: dungeon
scope:
  range:
    from:
      type: time
      item: T10
    to:
      type: time
      item: T15

id: hero-has-sword
type: SPATIAL
name: possesses
source: hero
target: magic-sword
scope:
  and:
  - type: frame
    item: alt-timeline
  - range:
      from:
        type: time
        item: T5
      to: null

Typeobject
RequiredNo

Defined inframe.schema.json

Description: An alternative timeline or branch. Frames form a graph via parent references and relations.

Examples:

id: main
name: Main Timeline

id: alt-timeline
name: What if the hero refused?
parent: main
branches_at: T42
relations:
- target: main
  type: BRANCHES_INTO
  at: T42
  description: Hero refuses the call

id: time-loop
name: Groundhog Day Loop
parent: main
relations:
- target: main
  type: LOOPS_BACK
  at: T100
properties:
  loop_count: 3
  observer: char-hero

6.1.2. Property constraint.schema.json

Typeobject
RequiredNo
Additional propertiesAny type allowed
Defined inconstraint.schema.json

Description: A world rule that validates the world graph.

Examples:

id: no-isolated-characters
name: Every character knows at least one other character
description: Every character must have at least one relationship with another character.
severity: WARNING
scope:
  type: node_type
  item: CHARACTER

id: magic-limit
name: Magic users can hold at most 3 spells
description: No person may hold more than three spells at any given time.
severity: ERROR

id: no-undead-before-war
name: Undead cannot exist before the Great War
description: The undead do not exist in the world before the Great War.
severity: ERROR
scope:
  and:
  - in:
    - type: node_type
      item: CHARACTER
    - type: node_type
      item: CREATURE
  - type: tag
    item: undead
  - not:
      in:
      - type: frame
        item: timeline-b
  - range:
      from:
        type: time
        item: T00
      to:
        type: time
        item: T50
